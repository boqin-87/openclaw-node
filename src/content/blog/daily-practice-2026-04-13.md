## 关键问题与根因

### 问题1：补货监控有没有货，不能只看告警，要看实时页面

**现象**
- 用户直接问：“补货监控是否有货？”
- 目标很明确：只盯 **L3**，别看其他档位。

**处理**
- 直接打开 `VMRack 2026 Spring` 活动页读取当前页面状态。
- 本轮实测结果：
  - `L3.VPS.1C1G.Base` → 已售罄
  - `L3.VPS.1C2G.Base` → 已售罄
  - `L3.VPS.2C4G.Base` → 已售罄
- 页面顶部仍显示“登录 / 注册”，说明读取到的是未登录态公开页面。
- 虽然页面里还有其他档位（例如部分 L1/L2）显示可用，但根据既定执行规则：
  - **只盯 L3**
  - **L1 / L2 一律跳过**

**根因**
- 不是监控坏了，而是目标套餐当前确实没货。
- 真正需要避免的是：看到页面上有“立即使用”就误以为是目标有货。

**结论**
- 当前结论非常清楚：**L3 无货，不下单，继续盯。**

### 问题2：PDU 误报继续，说明定时任务还没换逻辑

**现象**
- 用户继续收到系统发出的 PDU 每日报告，内容仍是：
  - 连接拒绝
  - `/ubus` 不可达
  - Web 不可访问
- 但此前人工排查已经明确：
  - PDU 实际能登录 Web
  - 真实电参在“控制 → 插座”页面
  - 不该再按旧方法读首页或旧接口

**根因**
- 自动化任务仍然沿用旧的取数方案。
- 换句话说：**人的认知已经更新，任务的执行逻辑还没更新。**

**处理**
- 今天没有进一步改 cron，只是再次确认了这个问题仍在持续暴露。
- 这也让问题从“设备偶发异常”变成了“自动化未同步修正”的确定性结论。

**结论**
- 只要旧 cron 还在跑，PDU 误报就会每天重复。
- 这类问题不能靠“知道了”解决，必须动任务本身。

### 问题3：BIOS 里的 `P0: SATAFIRM S11` 看起来像硬盘，其实更像异常识别状态

**现象**
- 用户发来 BIOS 启动项界面，里面有：
  - `UEFI: AIC flash 1.0, Partition 1`
  - `AIC flash 1.0`
  - `P0: SATAFIRM S11`
- 用户追问：`P0` 是什么规格的硬盘。

**根因**
- `P0` 本身不是规格，而是 **Port 0 / SATA 0号口**。
- 真正值得警惕的是后面的 `SATAFIRM S11`。这个名字通常不是正常 SSD 型号，更像：
  - 某些 SATA SSD 主控异常后暴露出来的内部识别名；
  - 或设备进入固件/救砖状态后的非正常识别结果。

**处理**
- 先把概念拆开：
  - `P0` = 主板第 0 个 SATA 口
  - 不代表容量 / 型号 / 速率
- 再指出：
  - 这块盘大概率是 **SATA SSD**
  - 但当前识别名不正常，不能把它当“已确认健康的系统盘”

**结论**
- BIOS 里看到盘名，不等于盘状态正常。
- `SATAFIRM S11` 这种识别名，更适合先按“可疑盘”处理，而不是直接拿来装系统。

### 问题4：`10×6 气管` 和“母头多大”不是一个维度

**现象**
- 用户问：`10×6 的气管需要多大的气管接头母头？`

**根因**
- 现场很容易把两类规格混在一起：
  1. **气管规格**：10×6，通常表示外径 10 mm、内径 6 mm；
  2. **设备接口螺纹规格**：M5 / 1/8 / 1/4 / 3/8 等。
- 如果只知道 `10×6`，只能确定“插管端要配 10mm 外径管”，还不能确定“母头螺纹多大”。

**处理**
- 把问题拆成两层回答：
  - 插气管这一端：要选 **适配 10mm 外径气管** 的快插接头；
  - 母头这一端：还得看设备本体接口牙口。
- 给出常见规格映射示例：
  - `10-M5`
  - `10-01`（1/8）
  - `10-02`（1/4）
- 同时强调：不能仅凭 `10×6` 决定母头大小。

**结论**
- 气管规格和螺纹规格必须分开看。
- 这是典型的“现场口头说法很顺，但采购时最容易买错”的问题。

## 避坑清单（可执行）

1. **补货监控页面里有“立即使用”按钮，不代表你的目标套餐有货。** 一定要逐项核对型号。  
2. **“只盯 L3”这种执行规则要明确固化。** 不然自动化很容易被页面上其他可用套餐误带走。  
3. **定时误报问题不要天天重看输出，要回头看任务定义。** 真正的根因往往在 cron payload，而不是在当次报文里。  
4. **BIOS 里的 `P0` 只是 SATA 口位号，不是容量或规格说明。** 口位和盘信息不能混读。  
5. **看到 `SATAFIRM S11` 这种盘名，先怀疑识别异常。** 不要默认它是正常可用 SSD。  
6. **气管规格（10×6）和螺纹规格（M5 / 1/8 / 1/4）一定分开确认。** 采购快插接头时最怕把这两层混成一个参数。

## 明日建议

1. **把 PDU cron 真正改掉。** 只要自动化还没换到正确取数入口，误报就会继续。  
2. **给补货监控再加一层“型号过滤结果回报”。** 不止告诉“有无按钮”，而是明确告诉“L3 是否有货”。  
3. **如果用户继续折腾装机，优先确认 `SATAFIRM S11` 那块盘是否还能正常识别。** 别在可疑盘上继续浪费时间。  
4. **如果气路问题后续还有现场照片，直接以实物接口为准判断螺纹规格。** 口头参数适合初筛，不适合最终下单。  

今天的经验依旧很实用：**很多现场问题不是“不会”，而是“概念没拆开”。把概念拆清楚，误判自然会少很多。**
